# gas-file-copy

Googleドライブで共有されているファイルを、メタデータ（名前、作成日、更新日）を保持したままマイドライブにコピーするGoogle Apps Script（GAS）です。

## 概要

- 共有されたファイル（コピー元）をマイドライブに直接コピー（複製）します。
- コピーの際、元のファイルが持っていた **「作成日」** と **「最終更新日」** をそのまま維持します。
- 新しく生成されたコピーファイルの所有権は、スクリプトを実行したユーザー（あなた）になります。
- GASの拡張サービスである「Drive API (v3)」を使用しています。

## 使い方

### 1. コピー元の準備（共有元のユーザー）

#### ① ファイルを共有する
共有したい（コピーさせたい）ファイルを、スクリプトを実行するアカウントに対して共有（権限は「閲覧者」または「編集者」）します。

---

### 2. コピー先の作業（スクリプト実行者）

#### ① Drive API を有効にする
GASのエディタ画面（Google Apps Script）で、以下の手順でDrive APIを追加します。

1. 画面左側メニューの「サービス」の横にある「**＋**」ボタン（サービスを追加）をクリックします。
2. リストの中から「**Drive API**」を探して選択します。
3. バージョンが「**v3**」になっていることを確認し、「追加」をクリックします。
4. 左メニューの「サービス」の下に「Drive」という文字が表示されれば準備完了です。

#### ② GASの設定（ファイルIDの入力）
コード内の設定項目（`SOURCE_FILE_ID`）に、共有された元のファイルの「ファイルID」を入力します。

> 💡 **ファイルIDの見分け方**
> ファイルIDはURL全体ではありません。URLの中の特定の英数字の文字列を指します。
>
> **【Googleドキュメントやスプレッドシートなどの場合】**
> `https://docs.google.com/spreadsheets/d/1aBcD2eFgHiJkLmNoPqRsTuVwXyZ1234567890/edit#gid=0`
> ➔ ファイルIDは `/d/` と `/edit` に挟まれた **`1aBcD2eFgHiJkLmNoPqRsTuVwXyZ1234567890`** の部分です。
>
> **【PDFや画像（jpg/png）、動画などのファイルの場合】**
> `https://drive.google.com/file/d/1xYzA2bCdEfGhIjKlMnOpQrStUvW9876543210/view?usp=sharing`
> ➔ ファイルIDは `/d/` と `/view` に挟まれた **`1xYzA2bCdEfGhIjKlMnOpQrStUvW9876543210`** の部分です。

#### ③ GASを実行する
設定完了後、エディタ上部の「実行」ボタンを押してスクリプトを動かします。
処理が完了すると、マイドライブのルート（一番上の階層）に、日付が維持された状態のコピーファイルが作成されます。

## 注意点

- 本スクリプトは**ファイル単体専用**です。フォルダのコピー（およびフォルダ内の階層構造の維持）には対応していません。

## 使用技術

- Google Apps Script (GAS)
- Google Drive API v3
